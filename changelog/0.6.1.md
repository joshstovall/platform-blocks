## Breaking Changes

### Layout Prop API Standardization
**Major prop naming changes across all components for consistency and brevity:**

- `width` → `w`
- `height` → `h`
- `maxWidth` → `maxW`
- `minWidth` → `minW`
- `maxHeight` → `maxH`
- `minHeight` → `minH`

**Affected components:** Image, Video, Waveform, Skeleton, Dialog, Menu, Popover, HoverCard, Select, TextArea, Table, QRCode, Spotlight, TimePickerInput, MonthPickerInput, YearPickerInput

**Migration:**
```tsx
// Before
<Image width={300} height={200} />
<Dialog maxWidth={600} />
<Select maxHeight={400} />

// After
<Image w={300} h={200} />
<Dialog maxW={600} />
<Select maxH={400} />
```

### Lottie Component Removed
- The `Lottie` component and all related dependencies have been removed from the library
- Removed `lottie-react`, `lottie-react-native`, and `@lottiefiles/dotlottie-react`
- This reduces the optional dependency footprint by ~28 MB
- **Migration:** Use a dedicated Lottie library like `lottie-react-native` directly if needed

### RichTextEditor Component Removed
- The `RichTextEditor` component has been removed from the library
- **Migration:** Use alternative rich text editing solutions or implement custom editors

## Features

### Column Component Improvements
- `Column` now defaults to `fullWidth={true}` since vertical layouts typically fill available width
- This eliminates the need to explicitly set `fullWidth` on most Column instances

### Select Component UX Improvements
- Added `matchAnchorWidth` option to position engine for dropdown inputs
- Select dropdown now matches trigger width by default for better visual consistency
- Improved selection indicator: moved checkmark icon to the left with consistent spacing
- Refined hover/active states for better visual feedback
- Updated highlight color scheme in dark mode for better contrast

### Timeline Component Enhancements
- Added `timestamp` prop to Timeline.Item for displaying time information
- Added `titleColor`, `descriptionColor`, and `timestampColor` props for granular text color control
- Both Timeline (global) and Timeline.Item (per-item) support these color overrides
- Improved text styling propagation through nested React elements

### Input Component Improvements
- Enhanced web input styling to prevent browser default outline/border interference
- Added comprehensive style resets for web platform: `outlineStyle: 'none'`, `borderWidth: 0`, `boxShadow: 'none'`, `boxSizing: 'border-box'`
- Eliminates double-border issues and inconsistent focus states across browsers

### Text Component Display Control
- Text component now respects explicit `display` and `whiteSpace` style overrides
- Default behavior remains `display: 'inline'` on web
- Allows for block-level text rendering when needed: `<Text style={{ display: 'block' }}>...</Text>`

### Documentation Improvements
- Cleaned up component markdown files: removed redundant title headers (already in frontmatter)
- Affected components: KeyCap, MonthPicker, MonthPickerInput

### ESLint 9 Migration
- Migrated from legacy `.eslintrc.cjs` to modern flat config (`eslint.config.js`)
- Updated to `typescript-eslint` v8+ with proper flat config support
- Added comprehensive plugin setup: react, react-hooks, react-native, unused-imports
- Removed deprecated `--ignore-path` and `--ext` flags from lint scripts

## Fixes

### Peer Dependencies
- **Added** `react-native-worklets` as a peer dependency (required by `react-native-reanimated` 4.x in Expo SDK 54+)
- Marked additional peer dependencies as optional in `peerDependenciesMeta`:
  - `@react-native-masked-view/masked-view`
  - `@shopify/flash-list`

  - `expo-linear-gradient`

- Fixed `useDeviceInfo` hook to handle the new `"unspecified"` color scheme value introduced in React Native 0.83
- Previously the hook expected only `"light"`, `"dark"`, or `null`, which caused TypeScript errors with newer RN versions
- Now properly maps `"unspecified"` to `"no-preference"`

### Universal CSS Enhancements
- Added comprehensive browser input resets to prevent default outline/border styling
- Applied resets to `input`, `textarea`, `select` elements and focusable React Native Web elements
- Ensures consistent cross-platform appearance without browser interference
- CSS now updates dynamically if changes are detected (previously only injected once)

### Dark Theme Highlight Colors
- Changed highlight color palette from amber/yellow to blue tones for better consistency
- Primary highlight now uses blue palette (aligned with brand colors)
- Updated text selection and highlight backgrounds to use primary blue with appropriate opacity
- Improved contrast and readability in dark mode

### Code Quality
- Removed unused ESLint disable directives across multiple files

- Removed deprecated usage of non-null assertion operators where possible

### Positioning Engine
- Added `matchAnchorWidth` option to overlay positioning calculations
- When enabled, overlay width matches anchor width without `maxWidth` constraints
- Integrated into cache key for proper memoization
- Improves dropdown input alignment and visual consistency

### Test Suite Updates

## Migration Guide

### Updating Layout Props
The most impactful change in this release is the prop naming standardization. Use find-and-replace to update your codebase:

```bash
# Example migrations
width=      → w=
height=     → h=
maxWidth=   → maxW=
minWidth=   → minW=
maxHeight=  → maxH=
minHeight=  → minH=
```

**Note:** This change affects all components that accept sizing props, including but not limited to: Image, Video, Waveform, Skeleton, Dialog, Menu, Popover, Select, TextArea, Table.

### Removing Lottie Usage
If you're using the `Lottie` component, you'll need to install and use `lottie-react-native` directly:

```bash
npm install lottie-react-native
```

### Removing RichTextEditor Usage
Consider alternatives like:
- Custom text input with basic formatting controls
- Third-party rich text libraries compatible with React Native
- Native TextInput with limited formatting

### Column fullWidth Behavior
If you were relying on `Column` NOT filling width by default, explicitly set `fullWidth={false}`:

```tsx
// If you need the old behavior
<Column fullWidth={false}>
  {children}
</Column>
```
