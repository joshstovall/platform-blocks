# 0.10.1

A small follow-up to 0.10.0 with component fixes and a few additive props. No breaking changes — everything here is backward compatible.

## Fixes

### Button now grows and shrinks correctly inside flex rows

`Button` renders its `Pressable` two `View`s deep, so `fullWidth`, width-family layout props (`w` / `maxW` / `minW`), and `flex*` styles applied to the inner element couldn't actually size the button within a flex container. Those props are now hoisted onto the outer wrapper, while visual styling (background, border, radius) stays on the `Pressable`. `fullWidth` buttons fill their parent and flex buttons distribute space as expected in a row. The measured-width freeze during `loading` is preserved.

### Toast icon centers on Android

The leading icon / loading spinner forced `alignSelf: 'flex-start'` on Android, pinning it to the top of the toast while web centered it. It now inherits the row's `alignItems: 'center'` on every platform for consistent vertical alignment.

### ThemeModeProvider `auto` mode follows the system theme on native

In `auto` mode, `systemColorScheme` was only wired up to the web `matchMedia` listener, so native apps were stuck at the initial value and never followed OS light/dark changes. The provider now seeds from `Appearance.getColorScheme()` synchronously on native (no light flash before hydration) and subscribes to `Appearance` changes so the theme tracks the OS.

### YouTubePlayer no longer warns "Cannot update a component while rendering"

The player kept hot playback values (current time, duration, volume, mute) in React state and updated them from the ~4×/sec polling messages, which triggered the *"Cannot update a component while rendering a different component"* warning and unnecessary re-renders. Those values now live in refs — read by the imperative `getState()` and re-emitted through callbacks — so the polling cadence drives zero re-renders. The `message` handler is now referentially stable (parent callbacks are read through refs), an explicit "Video Unavailable" error state is rendered on player errors, and leftover `console.log` debugging was removed.

## Additions

- **`Avatar` `fallback` accepts a React node.** Previously `string`-only (initials); it now also accepts a custom node such as an icon. String / empty values still render as before.
- **`YouTubePlayer` `playerBaseUrl` prop.** Override the hosted player page URL per instance (e.g. to self-host the iframe relay) instead of depending on the default domain.
- **`YouTubePlayer` `playbackRate` and `quality` props are now applied.** Both were accepted but never sent to the player; they're now pushed once it's ready and whenever the props change.
